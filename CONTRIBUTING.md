# Contributing
First off, thank you for wanting to make `chip8py` better! Anyone is completely welcome to contribute, submit bug fixes, or suggest new features.

## The Golden Rules
- **No massive formatting overhauls:** Please avoid sweeping formatting changes (like re-indenting entire files, changing all single quotes to double quotes, or running aggressive auto-formatters) unless it is strictly required for a functional fix. Let's keep git diffs clean and focused on actual code changes.
- **Keep PRs focused:** One feature or one bug fix per Pull Request is highly preferred.
- **Try opening an issue first:** If you aren't sure on how to fix something, you can open an issue for it instead. This also goes for feature suggestions. Even though I'm not very picky on what features to add, it'd suck if you spend your time implementing a new feature only to get it rejected.

## Quickstart
1. Fork the repo and create your branch from `main`.
2. Make your changes and test them locally.
3. Open a Pull Request with a brief description of what you changed or fixed.

That's all there is to it. Happy coding!
